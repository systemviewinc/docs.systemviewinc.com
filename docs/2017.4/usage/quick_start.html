Quick Start

Visual System Integrator: Quick Start

This Quick Start will describe how to create a simple system using Visual System Integrator.

1- Start the VSI software

To start the tool just type:

       1.1- >${VSI_INSTALL}/host//bin/vsi

               ${VSI_INSTALL} – path to VSI installation

               HOST_TYPE – linux.x86_64 | windows.x86_64

2- Import Existing platform

In this guide we will import an existing platform.

       2.1- Quick Start --> Open Example Project…

       2.2- Choose a name, location, and a default Xilinx board for your project.

       2.3- Choose zynq from platform_list.

               This will create a Platform with two “Execution Contexts” . A Software context “zynq_ps” and a Hardware context “zynq_pl”.

       2.4- Click Flow --> Compile Platform… to compile the platform.

3- Application System Canvas

       3.1- Clicking Flow --> Create System… to create an "Application Canvas".

               This will create a blank “System Canvas”.

       3.2- Clicking Flow --> Import Platform… to import the platform definition.

               After importing the platform, the execution contexts show up as blank entities.

4- Develop Application

       4.1- Import a Block From Library:

              4.1.1- Double click anywhere in the software context enter into it.

              4.1.2- Right-Click and select “Add IP… “.

              4.1.3- Select “VSI TCP Server v1.0”.

                      This will place the TCP/IP server in the System Canvas.

       4.2 Import C/C++ code

               We will use the “Software Import Wizard” to import a “sort” function as a block.

              4.2.1- Right-click and select “Add IP…”

              4.2.2- Select “VSI Software Import Wizard”.

              4.2.3- Double-Click on “vsi_gen_ip_0” to import the “sort” C function.

                     4.2.3.1- The “Source Directory” box either “Browse” to or enter the directory $(VSI_INSTALL)/target/common/hls_examples/sort

                                   The “Software Import Wizard” will call a “built-in” C/C++ parser and present drop-down list of functions it finds (C/C++ Function Name).

                     4.2.3.2- Choose the sort function, and navigate to the “Arguments” section.

                                   The “Arguments” section will show the two arguments “in_arr” and “out_arr” with some default values filled in.

                     4.2.3.3- Mark “in_arr” as an “Execution Trigger”; this indicates that the VSI runtime will execute this function whenever data is available in this input interface.

                     4.2.3.4- Mark the “Direction” of “out_arr” as “output”. The sort function puts the output into this buffer when it finishes execution.

                     4.2.3.5- Change “Access Type” of “out_arr” to “Random”. The sort algorithm accesses the output array in a non-sequential fashion.

                     4.2.3.6- Click OK. The interfaces of the block are now updated to reflect the name and direction specified in the “Arguments” section.

       4.3- Connect the “server_data_out” interface of the TCP/IP server block to the “in_arr” interface of the “vsi_gen_ip_0” block.

       4.4- Connect the “out_arr” interface of the “vsi_gen_ip_0” block to the “server_data_in” interface of the TCP/IP block to complete the System Design.

               Some blocks can be moved between execution contexts.

5- Build Application

       5.1- With the System Canvas open Click on Flow --> Generate System…

       5.2- Build the High-Level Synthesis blocks by clicking on Flow --> Build HLS…

       5.3- Build the executables for Software Contexts by clicking on Flow --> Build Software Contexts….

       5.4- Building the Hardware Project by clicking on Flow --> Build Hardware….

               This completes the build process for the example project.

6- Execute Application

The following steps showing how to execute the generated project on a Zynq-SoC Based MicroZed board.

       6.1- Prerequisites:

              6.1.1- Build the Linux kernel and root file system for the Board you are using

              6.1.2- Cloning repository https://github.com/systemviewinc/fpga_driver.git to your home directory

              6.1.3- Source the xilinx environment settings: source /opt/Xilinx/Vivado/< version>/settings64.sh

              6.1.4- Build the driver with make:make CROSS_COMPILE=arm-linux-gnueabihf- ARCH=arm KERNEL_SOURCES=/< home_dir>/build/driver_vsi/linux-xlnx

       6.2- Login to your board and create a directory in the root-file-system, this is an optional step. It will help you isolate the files being copied.

       6.3- Coppying the following files from the build directory to the target filesystem:

                            - The bitstream file. This file can be found at < hw_project_directory>/zynq_pl/zynq_pl.runs/impl_1/zynq_pl_wrapper.bin.1

                            - The driver installation script found at $< project_dir>/vsi_auto_gen/sw//driver.sh ($< project_dir>/vsi_auto_gen/sw/zynq_ps/driver.sh)

                            - The Software Executable $< project_dir>/vsi_auto_gen/sw/build//bin/ ($< project_dir>/vsi_auto_gen/sw/build/zynq_ps/bin/zynq_ps)

                            - The device driver “vsi_driver.ko” created in the previous step.

Footnotes
  1. For Yocto Linux kernels. The generated Bitstream file is zynq_pl_wrapper.bit that can be found at < hw_project_directory>/zynq_pl/zynq_pl.runs/impl_1/zynq_pl_wrapper.bit.
